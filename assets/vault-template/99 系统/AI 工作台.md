# AI 工作台

把以下指令发给库内的 Claudian/Codex。所有流程都必须遵守：先检索再创建、事实与推断分开、问题与答案分离、不得替用户勾选投递状态。

## 全局约束

1. 简历真实来源只有 [[07 简历库/00 简历事实底稿]]、[[07 简历库/01 当前通用简历]] 和用户本轮明确确认的事实。
2. 行为题答案只允许使用 [[04 答案库/故事库/_故事索引|行为故事库]] 中的真实故事；没有素材时创建“待补故事”的答案骨架，不能编造经历或数字。
3. 每个问题存入 `03 面试题库`，每个答案另存入 `04 答案库/题目答案`，两个页面双向链接。岗位页的 BQ 表中，题目文字直接链接答案；不再建立重复的“问题与答案页”。问题页仍保留练习记录，并在原文后放醒目的答案入口。
4. 新建问题、能力、答案前先全库语义去重；同义内容优先复用并追加来源。
5. [[01 岗位库/_投递清单]] 是唯一投递事实来源。`[x]` 表示已投递：不能再次推荐投递，但可以且应该用于后续面试训练。AI 不得自行勾选或取消勾选。
6. 创建真实笔记时只沿用模板字段与章节，不复制 `<% ... %>` Templater 代码。

## 流程 A：首次导入简历

```text
请使用 $obsidian-interview-growth-vault 建立我的简历事实底稿：
1. 读取我附加或 @提及的简历，以及 [[07 简历库/01 当前通用简历]]。
2. 原始 PDF/Word/图片保留在“07 简历库/原始文件”；把可解析正文整理到“01 当前通用简历”。
3. 按“仅简历”场景解析教育、工作、项目、技能、指标、行业、职级、个人贡献边界和数据来源，写入 [[07 简历库/00 简历事实底稿]]。
4. 所有事实标记“已确认 / 待确认 / 模型推断”。只有原简历明确内容可暂记为已确认；扩展解释和数字必须向我求证。
5. 不把参与升级为主导，不把团队结果归为个人结果，不补造工具、职责、证书或数据。
6. 最后一次只问一个最高价值的问题，等待我回答；不要一次给问卷。
```

## 流程 B：导入一个新 JD 并完整处理

```text
请完整处理我当前打开、附加、粘贴或 @提及的新 JD。

第一阶段｜防重复和原文归档
1. 先读取 [[01 岗位库/_投递清单]] 和 01 岗位库，按公司、岗位、JD 来源进行去重。
2. 如果它与某条已勾选岗位相同，明确提示“已经投递”，不要再次建议投递，也不要创建重复岗位页；可以更新该岗位的新信息并继续面试准备。
3. 如果是新岗位，按“99 模板/岗位模板.md”创建岗位页，保留完整 JD 原文，并在投递清单追加未勾选条目。AI 不得替我勾选。

第二阶段｜JD 拆解
4. 提取公司、岗位、岗位族、行业、职级、Base 地、现场/混合/远程、薪资范围、日薪（如适用）、薪资月数、奖金/股票/补贴和福利。保持原始薪资单位；信息缺失写“待确认”，不自行换算年包或日薪。
5. 将核心职责与能力要求合并成一张“JD 核心要求（职责 × 能力）”表，覆盖硬性门槛、核心技能、软技能、领域知识、工具平台、加分项、隐性偏好和淘汰风险；每项引用 JD 原文，避免重复展开。
6. 将能力链接到 02 能力库；不存在时按能力模板创建。

第三阶段｜简历匹配与更新方案
7. 使用 $obsidian-interview-growth-vault，读取简历事实底稿和当前通用简历，逐项输出“已覆盖 / 弱覆盖 / 未覆盖 / 不建议硬凑”，区分直接证据、可迁移证据和无证据。完整“JD-简历逐项映射”和评分依据只写入对应的定制简历页；岗位页只保留匹配结论、核心缺口和链接。
8. 只有同时具备可解析 JD 和简历时才给匹配分；评分必须附依据、扣分原因和补分动作，不代表录用率。
9. 在岗位页写入简历更新方案，并在“07 简历库/定制版本”创建该 JD 的独立版本。事实不足时状态为 needs-input，先提问，不生成伪最终版；不得覆盖通用简历或其他公司的版本。

第四阶段｜BQ 预测、问题与答案
10. 使用 $obsidian-interview-growth-vault，结合 JD Must Have、Hidden Signals、简历事实、故事库和 [[03 面试题库/_真题库索引|真题库]] 分析提问维度。证据依次参考：本人同公司/同岗位面试、同公司/同岗位外部真题、相似岗位真题、JD 与行业惯例。
11. 生成 Top 20 选题并按“必练 / 次重点 / 补充”分级。岗位页不展示“考察能力”，每道题只用纵向两列表格呈现三行：①题目 N（点击查看答案）②预测依据 ③可用真实故事。外部真题必须保留来源，不能写成公司官方题库；使用 high/medium/low 表示内部准备优先级，不伪造精确概率。先语义去重，同义题只追加岗位和来源。
12. 每道题在 03 面试题库创建/更新独立问题页，同时在 04 答案库/题目答案创建独立答案页并双向链接。岗位页题目文字直接链接答案页；表格内 Wiki 链接的别名分隔符必须转义为 `\|`，避免链接被拆成额外列。不得再生成单独的“问题与答案页”或重复 Top 5 列表。
13. 每个答案页必须依次包含回答策略、通用参考答案（适用时）、我的个性化答案、完整口述答案（2–3 分钟）、事实依据和追问准备。完整口述答案建议约 450–700 个汉字，必须是自然、连续、能直接说出口的完整句子：开头 15–20 秒给结论，中段 90–120 秒展开事实/案例/框架，结尾 20–30 秒回扣岗位或复盘；不能只是提纲或对前文的机械拼接。
14. 技术题/知识题可以生成通用参考答案；行为题和简历题只能用事实底稿与故事库生成个人答案。事实充分时必须生成完整口述稿并将 full_answer_status 标为 ready；缺少关键故事或数字时将其标为 needs-input，保留明确占位且一次只问一个关键事实，不能编造。
15. 将岗位页 analysis_status 更新为 ready；汇总 Base、薪资、匹配度、简历更新重点、Top 5 必练问题和待确认事项。
```

## 流程 C：录入一次真实面试并校准预测

```text
请整理我当前打开或粘贴的真实面试记录：
1. 按“99 模板/真实面试记录模板.md”保存到 05 面试记录，完整保留面试官原题、追问、我的原回答和卡点，不把 AI 改写冒充原话。
2. 每道原题先检索 03 面试题库。同义题复用原问题页，追加本人真实面试来源：`source_type=self-real`、面试记录、`self_occurrences + 1` 和 `last_seen`；不同考察点才新建问题页。
3. 每题必须在题目原文后直接链接独立答案页。若已有答案则基于本次追问更新；没有则在 04 答案库/题目答案创建。
4. 技术/知识题自行生成可验证的参考答案；行为题使用 $obsidian-interview-growth-vault 从真实故事库匹配答案，没有故事时创建待补骨架并一次只问我一个关键事实。事实充分时同时生成 2–3 分钟、约 450–700 字的完整口述答案。
5. 根据我的原回答给出结构、证据、岗位相关性、简洁度评分，更新下次复习日期。
6. 若问题此前在目标岗位预测表中，标记 `prediction_hit=true` 并保留当时的 `prediction_basis`；若未预测到，标记为“预测外新题”，加入未来同公司/同岗位族预测候选。不要因为某题本次没问就删除预测题。
7. 汇总本次面试官真正关注的能力、简历风险、预测命中题、预测外新题和下一轮训练动作，但不要删除原始记录。
```

## 流程 G：导入外部真题截图

```text
请处理我当前附加、粘贴或 @提及的小红书/牛客/脉脉等平台面试题截图：
1. 将图片保留到“98 附件/真题截图”，按“99 模板/真题来源模板.md”在“03 面试题库/真题来源”创建来源页；记录平台、原帖链接、发布者、发布日期、公司、岗位、轮次和收集日期。缺失写“待确认”。
2. 按截图顺序忠实 OCR；看不清的内容标“无法确认”，不猜字、不把 AI 改写当作原文。若当前图片无法访问，先让我重新附图。
3. 按不同考察点拆成独立问题，并全库语义去重。同一帖子多张截图只算一个外部独立来源；同义题复用问题页并追加 source_records/source_images。
4. 新题标记 source_type=external-real；填写 source_platform、source_company、source_role、external_occurrences、occurrence_count 和 last_seen。外部平台分享属于他人经验转述，不冒充公司官方题库。
5. 优先生成或更新每道题的独立答案页：技术/知识题给可验证的完整参考答案；行为题使用 $obsidian-interview-growth-vault，只能基于简历事实底稿与真实故事库。关键事实不足时标 full_answer_status=needs-input，一次只问我一个问题。
6. 每份答案都包含回答策略、通用参考答案（适用时）、我的个性化答案、2–3 分钟完整口述答案、事实依据和追问准备；问题页原文下方直接链接答案页。
7. 将处理后的题纳入 [[03 面试题库/_真题库索引]]。若已有相同公司/岗位的岗位页，按 [[99 系统/真题与预测校准规则]] 复核其必练/次重点/补充排序，但保留原 JD 预测依据和所有原始来源。
```

## 流程 F：生成或补全单道题答案

```text
请完善我当前打开或 @提及的问题及其独立答案页：
1. 先读取问题页的 category、考察能力、JD 来源和 answer_note；若没有答案页，按“99 模板/答案模板.md”创建并双向链接。
2. 把答案链接放在问题原文正下方，链接文字统一为“查看回答策略、个性化要点与 2–3 分钟完整答案 →”。问题页不得直接显示完整答案。
3. 答案页依次完成：回答策略、通用参考答案（适用时）、我的个性化答案、完整口述答案（2–3 分钟）、事实与故事依据、追问准备。
4. 完整口述答案建议 450–700 个汉字，使用自然口语和完整句子。开头直接回答问题，中段用事实案例或分析框架展开，结尾回扣目标岗位、业务价值或个人复盘；不得只是把策略提纲改成一段话。
5. 行为题/简历题必须使用 [[07 简历库/00 简历事实底稿]] 和 [[04 答案库/故事库/_故事索引|真实故事库]]。事实不足时不能补造；将 full_answer_status 设为 needs-input，并且一次只问我一个最高价值问题。
6. 技术题/知识题应给出完整、可验证的参考回答，并结合目标 JD 给出场景化示例；完成后将 full_answer_status 设为 ready。
7. 最后检查实际口述时长、重复表达、AI 腔、事实边界和可能追问。
```

## 流程 D：今日模拟面试

```text
请以当前 06 每日训练笔记记录一次今日模拟面试：
1. 先读取 [[01 岗位库/_投递清单]]。优先从已勾选（已投递）的岗位中选择需要准备的目标；若没有已投递岗位，再使用我指定的未投递岗位。
2. 读取该 JD、定制简历、03 面试题库、独立答案页、真实面试记录、故事库和最近 7 天训练。
3. 安排 2 道到期/薄弱旧题，并补充 2–3 道尚未覆盖的新题。新题必须先去重，问题页和答案页分开创建；在每日笔记中按“问题｜作答后查看答案”逐行配对，不建立独立答案清单。
4. 开始模拟后一次只问一道题，在我回答前禁止展示或暗示独立答案页内容。
5. 我回答后再读取答案和真实故事，按结构、事实证据、岗位相关性、表达简洁度各评 1–5 分，并继续追问一次。
6. 更新题目 review_count、last_score、last_reviewed、next_review；≤2 分明天复习，3 分三天后，4 分七天后，5 分十四天后。
7. 结束后完善每日笔记，并提醒我亲手勾选“完成一次模拟面试”。AI 不得替我勾选。
```

## 流程 E：每周自生长整理

```text
请执行本周知识库维护：
1. 按 role_family 汇总岗位共性职责、高频能力、Base 与薪资口径差异；只基于库内 JD。
2. 检查问题、答案、能力和故事的同义重复，优先合并链接与来源，不删除外部截图、真题来源页或真实面试原始记录。
3. 按本人真实面试、同公司同岗位外部真题、相似岗位真题、JD 预测四层证据更新题目优先级；统计预测命中、预测外新题、高频考察能力、尚无真实故事的行为题、低分题和过期复习题。
4. 更新首页和各索引；给出下周最值得补的 3 个真实经历/作品证据。
5. 不修改 [[01 岗位库/_投递清单]] 的勾选状态。
```
